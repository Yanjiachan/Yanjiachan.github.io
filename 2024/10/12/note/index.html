.proto生成.go

protoc --go_out=. --go_opt=paths=source_relative --go-grpc_out=. --go-grpc_opt=paths=source_relative helloworld.proto

–go_out：指定生成*.pb.go路径

–go-grpc_out：指定生成*_grpc.pb.go生成路径

–go_opt：参数，paths=source_relative相对路径

其中--proto_path=./proto用于指定 import 文件路径（默认为{$pwd}）

.proto生成.h .c

/opt/grpc/bin/protoc --grpc_out=. --cpp_out=. --plugin=protoc-gen-grpc=/opt/grpc/bin/grpc_cpp_plugin cube.proto

gRPC 中的 Service API 4种类型

  • UnaryAPI：普通一元方法，也就是普通API
  • ServerStreaming：服务端推送流
  • ClientStreaming：客户端推送流
  • BidirectionalStreaming：双向推送流

Stream 顾名思义就是一种流，可以源源不断的推送数据，很适合大数据传输，或者服务端和客户端长时间数据交互的场景。Stream API 和 Unary API 相比，因为省掉了中间每次建立连接的花费，所以效率上会提升一些。

最简单的方式

通过http://localhost:9091/metrics就可以获取到一些常见指标。其中

  • go_*：go运行时相关指标，如GC时间、goroutine数量
  • promhttp_*：工具包的相关指标，用于跟踪对指标请求的处理

添加自定义指标

  • 指标类型：
    • counter：只增不减计数器，描述累计状态，如访问量
    • gauge：可增可减计量器，描述当前状态，如内存使用率
    • histogram：直方图，描述分布情况，如请求响应时间，大于1s的有10w个，大于500ms的9w个，小于500ms的8w个
    • summary：类似直方图，但形式不同，如响应时间，30%大于1s，30%大于500ms，10%小于500ms

cmake 的find_package

find_package的查询路径
find_package包括设置查询路径和默认查询路径

  • 默认查询路径

    find_package查找.cmake的默认查询路径有如下

    • CMAKE_PREFIX_PATH
    • CMAKE_FRAMEWORK_PATH
    • CMAKE_APPBUNDLE_PATH
      echo $PATH可查询PATH包含哪些默认查询路径，以默认路径为根目录，find_package将查询目录下包含的.cmake文件

  • 设置查询路径

    让find_package在指定路径找依赖包，有三种方式

  1. 设置DIR

    set(Torch_DIR /home/libtorch)
    find_package(Torch REQUIRED)

  2. 设置PATHS

    find_package(Protobuf REQUIRED
    PATHS /home/libtorch
    NO_DEFAULT_PATH)

  3. 指定

    ​ set(CMAKE_PREFIX_PATH /home/libtorch)

注意CMAKE_INSTALL_PREFIX需要指定完整路径
